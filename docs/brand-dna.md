# Drawgle Brand DNA

## 1) Brand Core

### Positioning
Drawgle is the AI product-building workspace for teams who want speed without chaos. It combines guided planning, approved design tokens, and controlled generation so teams can create mobile app screens fast while keeping structure, consistency, and intent intact.

### Category
AI-powered product design and UI generation workspace (mobile-first).

### One-line Brand Promise
From idea to editable app screens, with control at every step.

### Mission
Help product teams ship production-ready mobile UI faster by turning prompts into governed generation workflows instead of one-shot AI output.

### Vision
Make AI-assisted app creation reliable enough for real product teams, not just demos.

## 2) Audience and Problem

### Primary Audience
- Product designers and PMs building new mobile experiences
- Startup teams validating app ideas quickly
- Agencies needing faster UI concept-to-delivery cycles

### Secondary Audience
- Founders who need a working visual product direction before engineering handoff
- Dev teams that want editable generated UI with design-system consistency

### Core Pains Drawgle Solves
- AI outputs look random or generic across screens
- Teams lose context between prompt, plan, and generation
- "Generate" flows hide decision points and create rework
- Editing one screen can break navigation and visual consistency
- Long AI runs feel opaque and hard to trust

## 3) Product Truths (What Drawgle Actually Does)

### Guided pre-build workflow
Users start in a protected workspace where they create the brief, generate design tokens, review screen plans, and only then build.

### Approved design system as build contract
Generation requires approved design tokens, preventing style drift and enforcing visual consistency.

### Plan-first, then build
Initial screen plans are reviewed before generation and passed directly into build flow.

### Real-time project memory
Context is assembled from project charter, design tokens, semantic screen retrieval, and message history so future outputs stay aligned.

### Unified agent narrative
Chat history shows user prompts, agent status updates, and applied edits in one project timeline.

### Safe editing with scoped changes
When a screen is selected, edits are scoped to likely blocks and audited before persistence, reducing regressions.

### Generation governance
Only one active run per project is allowed, reducing collisions and queue confusion.

### Navigation consistency enforcement
Planned navigation architecture and screen chrome rules are carried through planning, generation, and edits.

## 4) Value Pillars

1. Controlled speed
Move quickly, but through clear checkpoints: brief -> design system -> plan -> build.

2. Coherent output
Design tokens, charter context, and navigation rules keep screens aligned.

3. Transparent AI
Users can see status and decisions in chat rather than guessing what the agent is doing.

4. Editable by design
Generated screens are not dead mockups; they remain editable with structured safeguards.
